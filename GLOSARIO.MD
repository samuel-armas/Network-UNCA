# GLOSARIO CONCEPTUAL - NETWORK UNCA
**Materia:** Ingeniería de Software II

## 1. Conceptos de Gestión (Scrum)
* **Product Backlog:** Lista ordenada de todo lo que se conoce que es necesario en el producto. Es la única fuente de requisitos.
* **Sprint:** Periodo de tiempo fijo (time-box) durante el cual se crea un incremento de producto "Terminado", utilizable y potencialmente desplegable.
* **Definition of Done (DoD):** Acuerdo formal que define cuándo una Historia de Usuario se considera completa (incluye pruebas, revisión de código, etc.).

## 2. Conceptos de Desarrollo (Gitflow)
* **Main (Rama):** Rama que contiene el código de producción. Debe estar siempre estable y libre de errores.
* **Develop (Rama):** Rama de integración donde se fusionan las características nuevas. Es el "tronco" del desarrollo.
* **Feature Branch:** Rama temporal creada para desarrollar una funcionalidad específica sin afectar el código principal.
* **Pull Request (PR):** Petición para fusionar cambios de una rama a otra, permitiendo la revisión de código por pares antes de la integración.

## 3. Calidad y Pruebas
* **Prueba Unitaria (Unit Test):** Verificación automática de la parte más pequeña de código comprobable (un método o clase) de forma aislada.
* **Clean Code:** Práctica de escribir código que sea fácil de entender y fácil de mantener por otros desarrolladores (uso de nombres descriptivos, funciones pequeñas).
* **Refactorización:** Proceso de reestructurar el código existente sin cambiar su comportamiento externo, para mejorar su legibilidad y reducir la complejidad.
